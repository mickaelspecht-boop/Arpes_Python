1.2 ARCHITECTURE — POINTS CRITIQUES
-----------------------------------

a) **God class `ArpesExplorer` (app.py)** — Hérite QMainWindow. Stockage état
   global (raw_data, fit_res, sel_ev, sel_k, fit_selected, etc.).

b) **`_PROXY_MAP` dans ArpesExplorer** — dict `{method_name: controller_attr}`.
   `__getattr__(self, name)` redirige `self._foo()` vers
   `self._theory_overlay_ctrl._foo()`. Toute nouvelle méthode controller doit
   être ajoutée à `_PROXY_MAP` pour être appelable via window.

c) **Controllers** — classes avec `__init__(self, parent)`. Accèdent à
   `self._parent._session`, `self._parent._params`, `self._parent._status(msg)`,
   `self._parent._raw_data`, etc. Pas de QWidget direct dans controllers.

d) **PlotController est spécial** — utilise `__getattr__`/`__setattr__` pour
   proxy transparent vers `_parent`. `self._fit_res` accède à
   `parent._fit_res`. Ne pas casser ce pattern.

e) **Signals PyQt** — déclarés dans `FitParamsPanel` (params.py) en haut de
   classe. Wiring dans `arpes/ui/builders/panels.py:wire_param_signals`.
   Connexions canvas matplotlib dans `_connect_map_canvas`.

f) **Dataclasses frozen** — `FileEntry`, `FileMeta`, `FitParams` mutables
   (assigner direct). `TheoryBandData`, `TheoryOverlayConfig` frozen → recréer
   pour modifier (dataclasses.replace).

g) **Persistance Session** — `Session.save()` écrit
   `<folder>/.arpes_session.json`. Appelé après chaque modif état entry. Fields
   ajoutés à `FileEntry`/`FileMeta` doivent passer round-trip JSON
   (`asdict`/`from_dict`). `Session.to_payload()` / `load_from_payload()` pour
   sessions nommées portables.

h) **Plafond LOC** — pas de fichier > 700 LOC. Si dépasse → split en builders
   (`params_*.py`) ou controllers. Prévoir avant d'ajouter.

i) **Tests obligatoires** — toute nouvelle fonction pure dans
   `arpes/analysis/`, `arpes/io/`, `arpes/core/` doit avoir test pytest dans
   `tests/`. UI tests via `QApplication` + `QT_QPA_PLATFORM=offscreen`.

j) **Caveman comments interdits** — code commenté en français normal, docstrings
   triple-quote pour modules et fonctions publiques.

1.3 PATTERN AJOUT FEATURE STANDARD (template)
---------------------------------------------

Cas typique : nouveau bouton qui appelle un calcul puis met à jour UI.

  Step 1. Modèle (si nouveau type de donnée)
    → Ajouter champ à `FileEntry` ou `FileMeta` (dataclass mutable, default
      sensé pour rétrocompat session JSON existante).
    → Si frozen (TheoryBandData), créer nouvelle dataclass dans le module.

  Step 2. Logique pure
    → Module dans `arpes/analysis/` ou `arpes/io/` ou `arpes/core/`. Pas
      d'import PyQt. Inputs/outputs explicites.
    → Test pytest associé dans `tests/test_<module>.py`.

  Step 3. Controller
    → Ajouter méthode `_do_thing(self)` dans le controller pertinent.
    → Si nouveau controller : créer fichier dans `arpes/ui/controllers/`,
      class avec `__init__(parent)`, importer dans `app.py`, instancier dans
      `ArpesExplorer.__init__`.

  Step 4. Proxy
    → Ajouter `"_do_thing": "_my_ctrl"` au `_PROXY_MAP` dans app.py.

  Step 5. UI widget
    → Bouton/spinbox dans `params_*.py` ou nouveau `build_*` helper.
    → Signal `pyqtSignal()` dans `FitParamsPanel`.

  Step 6. Wiring signaux
    → `panels.py:wire_param_signals`: `p.thing_requested.connect(window._do_thing)`.

  Step 7. Smoke test
    → `python -c "import os; os.environ['QT_QPA_PLATFORM']='offscreen';
       from PyQt6.QtWidgets import QApplication; from arpes.app import
       ArpesExplorer; app=QApplication([]); w=ArpesExplorer(); ..."`.

  Step 8. Commit
    → `feat(<scope>): <description courte>` + body listant fichiers + raison.
